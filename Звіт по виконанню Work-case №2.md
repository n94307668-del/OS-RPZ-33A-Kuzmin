## 1. Встановлення гіпервізора II типу: Oracle VM VirtualBox

Для виконання цього завдання було встановлено гіпервізор II типу **Oracle VM VirtualBox**. Це потужний, безкоштовний продукт для віртуалізації з відкритим вихідним кодом, який дозволяє запускати декілька операційних систем на одному комп'ютері. Він підтримується на більшості сучасних операційних систем, включаючи Windows, macOS та Linux, що робить його універсальним інструментом для розробників, тестувальників та системних адміністраторів.

## 2. Опис базових дій у VirtualBox

### 2.1. Створення нової віртуальної машини

Процес створення  віртуальної машини (ВМ) у VirtualBox є інтуїтивно зрозумілим і складається з наступних кроків:

1.  **Запуск майстра створення:** У головному вікні VirtualBox необхідно натиснути кнопку "New" ("Створити").
2.  **Ім'я та операційна система:** Вказати ім'я віртуальної машини (наприклад, "Ubuntu Desktop"), обрати теку для її розміщення, а також тип та версію операційної системи, що буде встановлюватися. VirtualBox автоматично запропонує базові налаштування на основі обраного типу ОС.
3.  **Виділення оперативної пам'яті (RAM):** На цьому етапі необхідно визначити, скільки оперативної пам'яті хост-системи буде виділено для віртуальної машини. Рекомендований обсяг залежить від вимог гостьової ОС.
4.  **Створення віртуального жорсткого диска:** Можна створити новий віртуальний жорсткий диск, або використати існуючий. Рекомендується створювати динамічно виділений диск (Dynamically allocated), який буде займати місце на фізичному носії по мірі заповнення, а не одразу резервувати весь вказаний об'єм.

### 2.2. Вибір та додавання обладнання

-   **Система:** Налаштування порядку завантаження, чипсету, кількості процесорів (CPU) та увімкнення розширених можливостей, таких як PAE/NX та апаратна віртуалізація (VT-x/AMD-V).
-   **Дисплей:** Виділення відеопам'яті, налаштування кількості моніторів та увімкнення 2D/3D прискорення.
-   **Носії:** Підключення образів оптичних дисків (.iso) для встановлення ОС, а також додавання віртуальних жорстких дисків.

### 2.3. Налаштування мережі та підключення до Wi-Fi

VirtualBox пропонує декілька режимів мережевих адаптерів:

-   **NAT:** Найпростіший спосіб надати ВМ доступ до Інтернету. ВМ знаходиться за віртуальним маршрутизатором, що ускладнює прямий доступ до неї з хост-системи.
-   **Bridged Adapter (Мережевий міст):** ВМ підключається до мережі як повноцінний окремий пристрій, отримуючи власну IP-адресу від маршрутизатора. Це найкращий варіант для підключення до Wi-Fi мережі, оскільки ВМ буде бачити ту ж мережу, що і хост-комп'ютер.
-   **Internal Network (Внутрішня мережа):** Дозволяє створити ізольовану мережу між кількома ВМ.
-   **Host-only Adapter (Віртуальний адаптер хоста):** Створює мережу між хост-системою та ВМ, без доступу до зовнішньої мережі.

Для підключення до Wi-Fi, найпростіше обрати режим **Bridged Adapter** і вказати фізичний Wi-Fi адаптер хост-системи.

### 2.4. Робота з зовнішніми носіями (USB)

Для роботи з USB-пристроями у віртуальній машині необхідно встановити **VirtualBox Extension Pack**. Після цього можна:

1.  Перейти до налаштувань ВМ, у розділ "USB".
2.  Активувати контролер USB (рекомендується USB 2.0 або 3.0).
3.  Додати фільтр для конкретного USB-пристрою, який буде автоматично підключатися до ВМ при її запуску.

Було створено  віртуальну машину та встановлено на неї дистрибутив **Ubuntu 22.04 LTS** з графічною оболонкою GNOME. Процес встановлення включав:

## 4. Встановлення GNU/Linux без графічного інтерфейсу та порівняння оболонок

### 4.1. Встановлення мінімальної конфігурації

Було створено другу віртуальну машину та встановлено на неї **Debian 12 "Bookworm"** у мінімальній конфігурації. Під час встановлення, у розділі вибору програмного забезпечення, було знято всі позначки, окрім "standard system utilities", щоб отримати систему без графічного інтерфейсу.

### 4.2. Встановлення GNOME

Після завантаження в термінальний режим, було встановлено графічну оболонку GNOME за допомогою таких команд:

```bash
sudo apt update
sudo apt install gnome-shell gnome-terminal gnome-control-center nautilus -y
```

Після встановлення та перезавантаження, система завантажилася в графічний сеанс GNOME.

### 4.3. Встановлення та порівняння з XFCE

Додатково було встановлено другу графічну оболонку – **XFCE**:

```bash
sudo apt install xfce4 xfce4-goodies -y
```

Тепер при вході в систему можна обрати, яку графічну оболонку завантажити: GNOME або XFCE.

**Порівняння GNOME та XFCE:**

| Характеристика      | GNOME                                       | XFCE                                          |
| ------------------- | ------------------------------------------- | --------------------------------------------- |
| **Філософія**       | Сучасний, мінімалістичний, орієнтований на простоту використання | Традиційний, легкий, орієнтований на швидкість та низьке споживання ресурсів |
| **Споживання ресурсів** | Високе (RAM, CPU)                           | Низьке                                        |
| **Налаштування**    | Обмежені за замовчуванням, потребує розширень (Extensions) | Дуже гнучке, велика кількість налаштувань "з коробки" |
| **Зовнішній вигляд** | Модерний, з власною концепцією робочого столу (Activities Overview) | Класичний робочий стіл з панеллю та іконками |
| **Продуктивність**  | Може бути повільним на слабкому обладнанні | Дуже швидкий, ідеально підходить для старих комп'ютерів та ВМ |

**Висновок:**

-   **GNOME** є чудовим вибором для користувачів, які цінують сучасний дизайн та простоту, і мають достатньо потужне обладнання.
-   **XFCE** є ідеальним варіантом для тих, хто потребує максимальної швидкодії, низького споживання ресурсів та гнучкості в налаштуванні, навіть ціною більш консервативного зовнішнього вигляду.

**Conclusion:**

-   **GNOME** is an excellent choice for users who appreciate modern design and simplicity, and have sufficiently powerful hardware.
-   **XFCE** is an ideal option for those who require maximum speed, low resource consumption, and flexibility in customization, even at the cost of a more conservative appearance.

## 5. Glossary of English Terms

*   **Hypervisor (Гіпервізор):** Software that creates and runs virtual machines (VMs).
*   **Virtual Machine (Віртуальна машина, ВМ):** An emulation of a computer system. Virtual machines are based on computer architectures and provide functionality of a physical computer.
*   **Host System (Хост-система):** The physical computer on which the hypervisor and virtual machines run.
*   **Guest OS (Гостьова ОС):** The operating system installed inside a virtual machine.
*   **VirtualBox:** A free and open-source type-2 hypervisor from Oracle.
*   **NAT (Network Address Translation):** A networking mode in VirtualBox that allows a VM to access the internet through the host's network connection.
*   **Bridged Adapter (Мережевий міст):** A networking mode that connects a VM directly to the physical network, making it appear as a separate device on the network.
*   **USB Passthrough (Прокидання USB):** A feature that allows a virtual machine to directly access a USB device connected to the host system.
*   **Graphical User Interface (GUI) (Графічна оболонка):** A type of user interface that allows users to interact with electronic devices through graphical icons and visual indicators.
*   **GNOME:** A popular desktop environment for Unix-like operating systems, known for its modern design and user-friendliness.
*   **XFCE:** A lightweight desktop environment for Unix-like operating systems, designed for speed and low resource consumption.
