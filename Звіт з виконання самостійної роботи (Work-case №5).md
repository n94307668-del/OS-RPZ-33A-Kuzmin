# Звіт з виконання Work-case №5
**Тема:** Робота з периферійним обладнанням в ОС Linux

---

## 1. Теоретичні питання

### 1.1. Механізм роботи ОС Linux з периферійним обладнанням  (флешка та принтер)
В ОС Linux робота з обладнанням базується на принципі **"все є файлом"**. Кожен пристрій представлений у системі як спеціальний файл у директорії `/dev`.

*   **Флеш-накопичувачі:** Коли ви підключаєте флешку, підсистема ядра **udev** виявляє новий пристрій і створює для нього файл (`/dev/sdb1`). Для доступу до даних цей файл потрібно "примонтувати" до існуючої директорії.
*   **Принтери:** Для роботи з принтерами Linux використовує систему **CUPS** (Common Unix Printing System). Вона керує чергами друку, драйверами та мережевим доступом. Принтери зазвичай доступні через файли пристроїв типу `/dev/usb/lp0`.

### 1.2. Суть операції монтування

**Монтування (mounting)** — це процес підключення файлової системи зовнішнього пристрою (або розділу диска) до загального дерева каталогів ОС Linux.

*   **Суть:** ОС робить вміст пристрою доступним через певну папку (точку монтування).
*   **Для чого використовується:** Оскільки в Linux немає поняття "дисків" як окремих сутностей (як у Windows), монтування є єдиним способом отримати доступ до даних на зовнішніх носіях.
*   **Як використовується:** 
    *   Автоматично: Сучасні графічні оболонки (GNOME, KDE) монтують флешки в `/media/user/label`.
    *   Вручну: Команда `mount /dev/sdb1 /mnt/usb`.
    *   Постійно: Через файл конфігурації `/etc/fstab`.

### 1.3. Різниця між Linux та Windows при роботі з периферією

| Характеристика | ОС Windows | ОС Linux |
| :--- | :--- | :--- |
| **Ідентифікація** | Присвоюється літера диска (C:, D:, E:) | Монтується в дерево каталогів (напр., `/mnt/data`) |
| **Драйвери** | Часто потребують встановлення .exe/.msi файлів | Більшість драйверів вбудовані в ядро (kernel modules) |
| **Файлова система** | NTFS, FAT32, exFAT | ext4, XFS, Btrfs + підтримка Windows-систем |
| **Межа пристроїв** | Обмежена кількістю літер алфавіту (26) | Практично необмежена |

## 2. Практична частина: Методика виконання

### 2.1. Підключення до віртуальної машини (VirtualBox)
1.  Вставте флешку в фізичний ПК.
2.  У налаштуваннях ВМ перейдіть до розділу **USB**.
3.  Додайте "USB Filter" для вашого пристрою.
4.  Запустіть ВМ, і пристрій з'явиться всередині Linux.
![alt text](image-13.png)
### 2.2. Робота через графічний інтерфейс (GUI)
1.  Відкрийте файловий менеджер (напр., Nautilus).
2.  Натисніть на іконку флешки на бічній панелі (вона автоматично примонтується).
3.  Знайдіть файл `document1.txt`, натисніть правою кнопкою миші -> **Copy**.
4.  Перейдіть у папку `Documents` на ВМ -> **Paste**.
5.  Для друку: відкрийте файл, натисніть `Ctrl+P` та оберіть підключений принтер.

### 2.3. Робота через термінал (CLI)
1.  **Визначення пристрою:**
    ```bash
    lsblk
    ```
    (Припустимо, флешка — це `/dev/sdb1`).
2.  **Монтування:**
    ```bash
    sudo mkdir /mnt/my_usb
    sudo mount /dev/sdb1 /mnt/my_usb
    ```
3.  **Копіювання файлу:**
    ```bash
    cp /mnt/my_usb/document2.txt ~/Documents/
    ```
4.  **Друк файлу:**
    ```bash
    lp ~/Documents/document2.txt
    ```
5.  **Розмонтування:**
    ```bash
    sudo umount /mnt/my_usb
    ```

### 3.2. Conclusions 
> During this work-case, I have explored the fundamental mechanisms of peripheral device management in Linux. The concept of "everything is a file" and the mounting process provide a flexible and powerful way to handle hardware. Unlike Windows, which uses static drive letters, Linux offers a unified directory tree, making system administration more scalable. Practical skills in both GUI and CLI environments are essential for efficient Linux usage.
