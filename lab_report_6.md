## 2. Завдання попередньої підготовки

### 2.1. Призначення команд та їх параметри

#### `tar` (Tape Archive)

**Призначення:** Команда `tar` використовується для архівування файлів та каталогів в один файл, відомий як 
«tarball» або «tar-архів». Вона не стискає файли за замовчуванням, але часто використовується в поєднанні з іншими утилітами стиснення, такими як `gzip`, `bzip2` або `xz`.

**Основні параметри:**
*   `-c` (create): Створити новий архів.
*   `-x` (extract): Розпакувати файли з архіву.
*   `-t` (list): Переглянути вміст архіву.
*   `-v` (verbose): Детальний вивід, показує файли, що додаються або витягуються.
*   `-f` (file): Вказати ім'я файлу архіву.
*   `-z` (gzip): Стиснути або розпакувати архів за допомогою `gzip`.
*   `-j` (bzip2): Стиснути або розпакувати архів за допомогою `bzip2`.
*   `-J` (xz): Стиснути або розпакувати архів за допомогою `xz`.
*   `-r` (append): Додати файли до існуючого архіву.
*   `-u` (update): Додати лише новіші файли до архіву.

**Встановлення:** `tar` зазвичай встановлений за замовчуванням у більшості дистрибутивів Linux. Якщо ні, його можна встановити за допомогою менеджера пакетів: `sudo apt-get install tar` (для Debian/Ubuntu) або `sudo yum install tar` (для CentOS/RHEL).

#### `xz`

**Призначення:** Утиліта `xz` використовується для стиснення та розпакування файлів за допомогою алгоритму LZMA2. Вона забезпечує дуже високий ступінь стиснення, що робить її ідеальною для архівування файлів, де розмір є критичним, але за рахунок більшого часу стиснення.

**Основні параметри:**
*   `-z` (compress): Стиснути файл (дія за замовчуванням).
*   `-d` (decompress): Розпакувати файл.
*   `-k` (keep): Зберігати вхідні файли після стиснення/розпакування.
*   `-l` (list): Вивести інформацію про стиснений файл.
*   `-c` (stdout): Вивести результат на стандартний вивід.
*   `-0` до `-9`: Рівень стиснення (0 - найшвидший, 9 - найсильніший, за замовчуванням 6).
*   `-e` (extreme): Використовувати екстремальний режим стиснення (може зайняти дуже багато часу).

**Встановлення:** `xz-utils` зазвичай встановлений. Якщо ні: `sudo apt-get install xz-utils` або `sudo yum install xz-utils`.

#### `zip`

**Призначення:** `zip` - це утиліта для архівування та стиснення файлів, яка широко використовується в різних операційних системах (Linux, Windows, macOS). Вона створює архіви у форматі `.zip`, які можуть містити кілька файлів та каталогів.

**Основні параметри:**
*   `-r` (recurse-paths): Рекурсивно включати каталоги.
*   `-u` (update): Оновити існуючі файли в архіві або додати нові.
*   `-d` (delete): Видалити файли з архіву.
*   `-l` (list): Вивести список файлів в архіві.
*   `-e` (encrypt): Зашифрувати архів.
*   `-0` до `-9`: Рівень стиснення (0 - без стиснення, 9 - максимальне стиснення, за замовчуванням -6).

**Встановлення:** `sudo apt-get install zip unzip` або `sudo yum install zip unzip`.

#### `bzip2`

**Призначення:** `bzip2` - це утиліта для стиснення файлів, яка використовує алгоритм Барроуза-Вілера. Вона забезпечує кращий ступінь стиснення, ніж `gzip`, але є повільнішою. Файли, стиснені `bzip2`, мають розширення `.bz2`.

**Основні параметри:**
*   `-z` (compress): Стиснути файл (дія за замовчуванням).
*   `-d` (decompress): Розпакувати файл.
*   `-k` (keep): Зберігати вхідні файли.
*   `-s` (small): Використовувати менше пам'яті під час стиснення, але з меншим ступенем стиснення.
*   `-1` до `-9`: Рівень стиснення (1 - найшвидший, 9 - найсильніший, за замовчуванням 9).

**Встановлення:** `sudo apt-get install bzip2` або `sudo yum install bzip2`.

#### `gzip`

**Призначення:** `gzip` - це найпоширеніша утиліта для стиснення файлів у Linux, яка використовує алгоритм DEFLATE. Вона швидша за `bzip2` та `xz`, але забезпечує менший ступінь стиснення. Файли, стиснені `gzip`, мають розширення `.gz`.

**Основні параметри:**
*   `-z` (compress): Стиснути файл (дія за замовчуванням).
*   `-d` (decompress): Розпакувати файл (еквівалентно команді `gunzip`).
*   `-k` (keep): Зберігати вхідні файли.
*   `-r` (recursive): Рекурсивно стискати файли в каталогах.
*   `-l` (list): Вивести інформацію про стиснений файл.
*   `-c` (stdout): Вивести результат на стандартний вивід.
*   `-1` (fast) до `-9` (best): Рівень стиснення (1 - найшвидший, 9 - найсильніший, за замовчуванням 6).

**Встановлення:** `gzip` зазвичай встановлений за замовчуванням. Якщо ні: `sudo apt-get install gzip` або `sudo yum install gzip`.

### 2.2. Три приклади реалізації архівування та стискання даних різними командами

1.  **Створення `tar.gz` архіву (архівування та стиснення за допомогою `tar` та `gzip`):**
    ```bash
    tar -czvf my_archive.tar.gz /path/to/directory
    ```
    Ця команда створює архів `my_archive.tar.gz` з вмісту `/path/to/directory`, використовуючи `tar` для архівування (`-c`) та `gzip` для стиснення (`-z`). Параметр `-v` забезпечує детальний вивід, а `-f` вказує ім'я файлу архіву.

2.  **Створення `tar.bz2` архіву (архівування та стиснення за допомогою `tar` та `bzip2`):**
    ```bash
    tar -cjvf my_archive.tar.bz2 /path/to/directory
    ```
    Ця команда аналогічна попередній, але використовує `bzip2` для стиснення (`-j`), що зазвичай призводить до меншого розміру файлу, але займає більше часу.

3.  **Створення `zip` архіву (архівування та стиснення за допомогою `zip`):**
    ```bash
    zip -r my_archive.zip /path/to/directory
    ```
    Ця команда створює `zip` архів `my_archive.zip` з вмісту `/path/to/directory`. Параметр `-r` забезпечує рекурсивне додавання каталогів.

### 2.3. Призначення команд `cat`, `less`, `more`, `head` та `tail`

#### `cat` (Concatenate)

**Призначення:** Команда `cat` (від 
«concatenate») використовується для виведення вмісту файлів на стандартний вивід, об'єднання кількох файлів в один, а також для створення нових файлів.

**Основні параметри:**
*   `-b` (number-nonblank): Нумерувати лише непорожні рядки.
*   `-n` (number): Нумерувати всі вихідні рядки.
*   `-s` (squeeze-blank): Стискати кілька порожніх рядків до одного.
*   `-E` (show-ends): Показувати символ `$` в кінці кожного рядка.
*   `-T` (show-tabs): Показувати символи табуляції як `^I`.

**Встановлення:** `cat` є базовою утилітою Unix і зазвичай встановлена за замовчуванням у всіх дистрибутивах Linux.

#### `less`

**Призначення:** `less` - це програма для перегляду текстових файлів, яка дозволяє користувачеві переглядати вміст файлу посторінково, рухаючись як вперед, так і назад. Вона не завантажує весь файл у пам'ять, що робить її ефективною для великих файлів.

**Основні параметри:**
*   `g`: Перейти до початку файлу.
*   `G`: Перейти до кінця файлу.
*   `/pattern`: Шукати `pattern` вперед.
*   `?pattern`: Шукати `pattern` назад.
*   `n`: Перейти до наступного збігу.
*   `N`: Перейти до попереднього збігу.
*   `q`: Вийти з `less`.

**Встановлення:** `less` зазвичай встановлена за замовчуванням. Якщо ні: `sudo apt-get install less` або `sudo yum install less`.

#### `more`

**Призначення:** `more` - це програма для перегляду текстових файлів, схожа на `less`, але з більш обмеженими можливостями навігації (переважно лише вперед). Вона також відображає вміст файлу посторінково.

**Основні параметри:**
*   `Spacebar`: Перегорнути на одну сторінку вперед.
*   `Enter`: Перегорнути на один рядок вперед.
*   `q`: Вийти з `more`.

**Встановлення:** `more` є базовою утилітою Unix і зазвичай встановлена за замовчуванням.

#### `head`

**Призначення:** Команда `head` використовується для виведення перших кількох рядків (за замовчуванням 10) файлу на стандартний вивід. Це корисно для швидкого перегляду початку файлу без необхідності відкривати його повністю.

**Основні параметри:**
*   `-n N` або `-N`: Вивести перші `N` рядків.
*   `-c N` або `-N`: Вивести перші `N` байтів.

**Встановлення:** `head` є частиною пакета `coreutils` і зазвичай встановлена за замовчуванням.

#### `tail`

**Призначення:** Команда `tail` використовується для виведення останніх кількох рядків (за замовчуванням 10) файлу на стандартний вивід. Це особливо корисно для моніторингу лог-файлів у реальному часі.

**Основні параметри:**
*   `-n N` або `-N`: Вивести останні `N` рядків.
*   `-c N` або `-N`: Вивести останні `N` байтів.
*   `-f` (follow): Виводити нові рядки, що додаються до файлу, у реальному часі (для моніторингу логів).

**Встановлення:** `tail` є частиною пакета `coreutils` і зазвичай встановлена за замовчуванням.

### 2.4. Принципи роботи командної оболонки з каналами, потоками та фільтрами

**Потоки (Streams):** У Linux кожна програма має три стандартні потоки вводу/виводу:
*   **Standard Input (stdin)** (дескриптор файлу 0): Канал, через який програма отримує вхідні дані. Зазвичай це клавіатура.
*   **Standard Output (stdout)** (дескриптор файлу 1): Канал, через який програма виводить свої звичайні результати. Зазвичай це екран терміналу.
*   **Standard Error (stderr)** (дескриптор файлу 2): Канал, через який програма виводить повідомлення про помилки. Зазвичай це також екран терміналу.

**Перенаправлення (Redirection):** Це механізм, який дозволяє змінювати джерело або призначення стандартних потоків. Замість того, щоб вхідні дані надходили з клавіатури, їх можна перенаправити з файлу. Аналогічно, вихідні дані або помилки можна перенаправити у файл замість виведення на екран.

*   `>`: Перенаправляє `stdout` команди у файл, перезаписуючи його вміст, якщо файл існує. (Наприклад, `command > file`)
*   `>>`: Перенаправляє `stdout` команди у файл, додаючи вміст до кінця файлу, якщо файл існує. (Наприклад, `command >> file`)
*   `<`: Перенаправляє вміст файлу на `stdin` команди. (Наприклад, `command < file`)
*   `2>`: Перенаправляє `stderr` команди у файл. (Наприклад, `command 2> file`)
*   `&>`: Перенаправляє як `stdout`, так і `stderr` команди у файл. (Наприклад, `command &> file`)
*   `2>&1`: Перенаправляє `stderr` у те саме місце, куди перенаправляється `stdout`. (Наприклад, `command > file 2>&1`)

**Канали (Pipes `|`):** Канали дозволяють з'єднувати стандартний вивід однієї команди зі стандартним вводом іншої команди. Це створює послідовність команд, де вихід однієї команди стає входом для наступної, дозволяючи обробляти дані поетапно. Синтаксис: `command1 | command2 | command3`.

**Фільтри (Filters):** Фільтри - це програми, які читають дані зі стандартного вводу, обробляють їх певним чином і записують результат на стандартний вивід. Багато команд Linux, такі як `grep`, `sort`, `cut`, `tr`, `wc`, є фільтрами, оскільки вони призначені для обробки потоків даних. Вони часто використовуються в поєднанні з каналами для створення складних конвеєрів обробки даних.

### 2.5. Призначення команди `grep`

**Призначення:** Команда `grep` (від "Global Regular Expression Print") є потужною утилітою командного рядка, яка використовується для пошуку рядків, що відповідають певному шаблону (регулярному виразу), у файлах або в стандартному вводі. Вона виводить рядки, які містять збіг.

**Основні параметри:**
*   `-i` (ignore-case): Ігнорувати регістр символів під час пошуку.
*   `-v` (invert-match): Інвертувати збіг, виводити рядки, які *не* відповідають шаблону.
*   `-c` (count): Вивести лише кількість рядків, що відповідають шаблону.
*   `-n` (line-number): Вивести номер рядка разом з відповідним рядком.
*   `-r` (recursive): Рекурсивно шукати в каталогах.
*   `-E` (extended-regexp): Використовувати розширені регулярні вирази (еквівалентно `egrep`).
*   `-w` (word-regexp): Шукати лише повні слова.
*   `--color`: Виділяти збіги кольором.

### 3.1. Опрацювання прикладів команд з лабораторних робіт 9 та 10

Наступна таблиця містить опис команд, виконаних під час лабораторних робіт 9 та 10, а також їх функціональність та результати.

| Назва команди | Її призначення та функціональність | Результат виконання |
|---|---|---|
| `mkdir mybackups` | Створення нової директорії `mybackups` у домашньому каталозі користувача. | Директорія `mybackups` створена. |
| `tar -cvf mybackups/udev.tar /etc/udev` | Команда `tar` використовується для об’єднання кількох файлів в один файл. ### 3.2. Команди `tar` для архівування та стиснення

#### Створення файлу з розширенням `.tar`
```bash
tar -cvf archive1.tar lab6_test/f1.txt
```
*   **Призначення:** Створює архів `archive1.tar`, що містить лише файл `f1.txt`.

#### Створення файлу з розширенням `.tar`, що складається з декількох файлів і каталогів одночасно
```bash
tar -cvf archive2.tar lab6_test/f1.txt lab6_test/f2.txt lab6_test/subdir
```
*   **Призначення:** Створює архів `archive2.tar`, що містить файли `f1.txt`, `f2.txt` та весь каталог `subdir`.

#### Перегляд вмісту файлу
```bash
tar -tvf archive2.tar
```
*   **Призначення:** Виводить список файлів та каталогів, що містяться в `archive2.tar`.

#### Витягнення вмісту файлу `tar`
```bash
mkdir -p extract_tar
tar -xvf archive2.tar -C extract_tar
```
*   **Призначення:** Розпаковує вміст `archive2.tar` у newly created directory `extract_tar`.

#### Створення архівного файлу `tar`, стисненого за допомогою `bzip2`
```bash
tar -cjvf archive.tar.bz2 lab6_test
```
*   **Призначення:** Створює архів `archive.tar.bz2` з каталогу `lab6_test`, стискаючи його за допомогою `bzip2`.

#### Витягнення вмісту файлу `tar bzip2`
```bash
mkdir -p extract_bz2
tar -xjvf archive.tar.bz2 -C extract_bz2
```
*   **Призначення:** Розпаковує вміст `archive.tar.bz2` у newly created directory `extract_bz2`.

#### Створення архівного файлу `tar`, стисненого за допомогою `gzip`
```bash
tar -czvf archive.tar.gz lab6_test
```
*   **Призначення:** Створює архів `archive.tar.gz` з каталогу `lab6_test`, стискаючи його за допомогою `gzip`.

#### Витягнення вмісту файлу `tar gzip`
```bash
mkdir -p extract_gz
tar -xzvf archive.tar.gz -C extract_gz
```
*   **Призначення:** Розпаковує вміст `archive.tar.gz` у newly created directory `extract_gz`.

### 3.3. Перенаправлення потоків виведення в Bash

| Команда | Що виконує команда? |
|---|---|
| `cmd 1> file` | Перенаправляє стандартний вивід (`stdout`) команди `cmd` у файл `file`, перезаписуючи його вміст. |
| `cmd > file` | Еквівалентно `cmd 1> file`. Перенаправляє стандартний вивід (`stdout`) команди `cmd` у файл `file`, перезаписуючи його вміст. |
| `cmd 2> file` | Перенаправляє стандартний вивід помилок (`stderr`) команди `cmd` у файл `file`, перезаписуючи його вміст. |
| `cmd >> file` | Перенаправляє стандартний вивід (`stdout`) команди `cmd` у файл `file`, додаючи вміст до кінця файлу. |
| `cmd &> file` | Перенаправляє як стандартний вивід (`stdout`), так і стандартний вивід помилок (`stderr`) команди `cmd` у файл `file`, перезаписуючи його вміст. |
| `cmd > file 2>&1` | Перенаправляє стандартний вивід (`stdout`) команди `cmd` у файл `file`, а потім перенаправляє стандартний вивід помилок (`stderr`) у те саме місце, куди був перенаправлений `stdout` (тобто у `file`). |
| `cmd >> file 2>&1` | Перенаправляє стандартний вивід (`stdout`) команди `cmd` у файл `file`, додаючи вміст до кінця файлу, а потім перенаправляє стандартний вивід помилок (`stderr`) у те саме місце, куди був перенаправлений `stdout` (тобто у `file`, додаючи до кінця). |
| `cmd 2>&1 > /dev/null` | Перенаправляє стандартний вивід помилок (`stderr`) у те саме місце, куди був перенаправлений стандартний вивід (`stdout`), а потім перенаправляє стандартний вивід (`stdout`) у `/dev/null` (тобто, і `stdout`, і `stderr` будуть відкинуті). |
| `cmd 2> /dev/null` | Перенаправляє стандартний вивід помилок (`stderr`) команди `cmd` у `/dev/null` (тобто, помилки будуть відкинуті). Стандартний вивід (`stdout`) буде виведено на екран. |
| `cmd1 | cmd2` | Вивід стандартного виводу (`stdout`) команди `cmd1` передається як стандартний ввід (`stdin`) команді `cmd2` через канал (pipe). |
| `cmd1 2>&1 | cmd2` | Перенаправляє стандартний вивід помилок (`stderr`) команди `cmd1` у стандартний вивід (`stdout`) `cmd1`, а потім обидва потоки (`stdout` та `stderr` `cmd1`) передаються як стандартний ввід (`stdin`) команді `cmd2` через канал. |

| `find -name '*.txt' > file.txt 2> /dev/null` | Шукає всі файли з розширенням `.txt` у поточній директорії та її підкаталогах. Список знайдених файлів записує у `file.txt`. Повідомлення про помилки (наприклад, "Permission denied") відкидаються (`/dev/null`). | Стандартний вивід (`stdout`) у файл, стандартний вивід помилок (`stderr`) у `/dev/null`. |
| `cat file1_unsorted | sort > file2_sorted` | Зчитує вміст `file1_unsorted`, передає його через канал команді `sort`, яка сортує дані, а потім записує відсортований результат у `file2_sorted`. | Канал (pipe) для `stdout` `cat` до `stdin` `sort`, `stdout` `sort` у файл. |
| `cat myfile | grep student | wc -l` | Зчитує вміст `myfile`, передає його команді `grep`, яка фільтрує рядки, що містять слово "student", а потім передає відфільтровані рядки команді `wc -l`, яка підраховує кількість рядків. | Два канали (pipes) для `stdout` `cat` до `stdin` `grep`, `stdout` `grep` до `stdin` `wc -l`. |

### 4.1. Порівняльна характеристика процесів стискання та архівування

| **Результат** | Один файл (архів), що містить оригінальні файли та їх структуру. Розмір архіву зазвичай дорівнює або трохи більший за суму розмірів оригінальних файлів. | Один файл меншого розміру, що містить закодовані дані. Оригінальний файл зазвичай замінюється стисненим. |
| **Збереження даних** | Завжди без втрат (lossless). Оригінальні файли можуть бути повністю відновлені. | Може бути без втрат (lossless) або із втратами (lossy). Без втрат: повне відновлення даних. Із втратами: деякі дані втрачаються для досягнення більшого стиснення. |

| **Комбіноване використання** | Часто архівування та стиснення комбінуються (наприклад, `tar.gz`, `tar.bz2`, `tar.xz`) для отримання одного стисненого архіву. | Стиснення може застосовуватися до окремих файлів або до вже створених архівів. |

### 4.2. Інші програми для стискання та архівування в Linux

Окрім `gzip`, `bzip2`, `xz`, `zip` та `tar`, в ОС Linux існують інші утиліти для стиснення та архівування:

*   **`7z` / `7za` (p7zip):** Реалізація архіватора 7-Zip для Unix-подібних систем. Підтримує високий ступінь стиснення, різні формати архівів (7z, ZIP, GZIP, BZIP2, TAR) та шифрування. Часто забезпечує краще стиснення, ніж `zip` або `gzip`.
    *   **Встановлення:** `sudo apt-get install p7zip-full`
    *   **Приклад:** `7z a archive.7z files_to_archive` (створити архів)

*   **`rar` / `unrar`:** Утиліти для роботи з архівами RAR. `unrar` зазвичай доступний для розпакування, тоді як `rar` для створення архівів є пропрієтарним програмним забезпеченням.
    *   **Встановлення:** `sudo apt-get install unrar`
    *   **Приклад:** `unrar x archive.rar` (розпакувати архів)

*   **`lzip`:** Утиліта для стиснення даних, що використовує алгоритм LZMA. Схожа на `xz`, але з акцентом на цілісність даних та швидкість декомпресії. Файли мають розширення `.lz`.
    *   **Встановлення:** `sudo apt-get install lzip`
    *   **Приклад:** `lzip file.txt`

*   **`zstd` (Zstandard):** Швидкий алгоритм стиснення без втрат, розроблений Facebook. Пропонує дуже високу швидкість стиснення/розпакування при хорошому ступені стиснення, часто перевершуючи `gzip` та `bzip2`.
    *   **Встановлення:** `sudo apt-get install zstd`
    *   **Приклад:** `zstd file.txt`

### 4.3. Порівняння алгоритмів стискання

| Алгоритм/Утиліта | Ступінь стиснення | Швидкість стиснення | Швидкість розпакування | Використання пам'яті | Призначення |
|---|---|---|---|---|---|
| **`gzip` (DEFLATE)** | Середній | Висока | Дуже висока | Низька | Загальне стиснення, швидка передача даних, сумісність. |
| **`bzip2` (Burrows-Wheeler)** | Високий | Низька | Середня | Середня | Краще стиснення, ніж `gzip`, для архівів, які зберігаються довше. |
| **`xz` (LZMA2)** | Дуже високий | Дуже низька | Середня/Висока | Висока | Максимальне стиснення для довгострокового зберігання та розповсюдження програмного забезпечення. |
| **`zip` (DEFLATE)** | Середній | Середня | Середня | Низька | Кросплатформне архівування та стиснення, зручність використання. |
| **`7z` (LZMA/LZMA2)** | Найвищий | Дуже низька | Середня | Висока | Максимальне стиснення, підтримка багатьох форматів, шифрування. |
| **`zstd` (Zstandard)** | Високий | Дуже висока | Дуже висока | Середня | Оптимальний баланс між швидкістю та ступенем стиснення, сучасна альтернатива `gzip`. |

**Найшвидший та найефективніший:**
*   **Найшвидший:** З точки зору швидкості стиснення та розпакування, **`zstd`** часто вважається найшвидшим серед сучасних алгоритмів, особливо при збереженні хорошого ступеня стиснення. **`gzip`** також дуже швидкий, особливо при розпакуванні.
*   **Найефективніший (за ступенем стиснення):** **`7z`** (з алгоритмом LZMA/LZMA2) та **`xz`** (LZMA2) забезпечують найвищий ступінь стиснення, що робить їх найефективнішими для економії дискового простору, хоча і за рахунок значно довшого часу стиснення та більшого використання пам'яті.

Вибір "найкращого" залежить від конкретних потреб: для швидкого архівування та передачі даних краще підійде `gzip` або `zstd`; для максимальної економії місця при довгостроковому зберіганні - `xz` або `7z`.

### 4.4. Програмні засоби для стискання та архівування у мобільному телефоні

У мобільних телефонах (Android, iOS) для стискання та архівування даних використовуються різні програмні засоби, часто інтегровані в файлові менеджери або доступні як окремі додатки:

*   **Вбудовані файлові менеджери:** Багато сучасних мобільних ОС мають вбудовані файлові менеджери, які підтримують базові операції з ZIP-архівами (створення, розпакування). Наприклад, "Файли" на Android або "Файли" на iOS дозволяють розпаковувати ZIP-архіви без сторонніх додатків.

*   **Сторонні додатки-архіватори:** Існують численні додатки, які надають розширені можливості:
    *   **RAR (Android/iOS):** Офіційний додаток від розробників WinRAR, підтримує створення та розпакування RAR, ZIP, TAR, GZ, BZ2, XZ, 7z, ISO, ARJ архівів. Дозволяє створювати зашифровані архіви.
    *   **ZArchiver (Android):** Популярний безкоштовний архіватор, що підтримує широкий спектр форматів (7z, zip, bzip2, gzip, xz, tar, rar, iso та інші). Має розширені функції, такі як часткове розпакування, багатотомні архіви.
    *   **iZip (iOS):** Додаток для iOS, що дозволяє керувати ZIP-файлами, включаючи стиснення, розпакування, шифрування та обмін файлами.
    *   **WinZip (Android/iOS):** Відомий архіватор, доступний на мобільних платформах, підтримує ZIP, ZIPX, 7z, RAR та CBZ формати. Пропонує функції шифрування та інтеграцію з хмарними сховищами.

Ці додатки дозволяють користувачам мобільних пристроїв ефективно керувати стисненими файлами, економити місце та обмінюватися даними.

### 4.5. Програмні засоби для стискання та (де)архівування даних у ОС сімейства Windows

У ОС Windows для стискання та архівування даних використовуються як вбудовані засоби, так і численні сторонні програми:

*   **Вбудовані засоби Windows (ZIP-архіви):** Windows має вбудовану підтримку ZIP-архівів. Користувачі можуть створювати ZIP-папки, додавати до них файли, розпаковувати їх без встановлення додаткового програмного забезпечення. Це зручно для базових операцій, але не підтримує інші формати архівів.

*   **Сторонні програми:**
    *   **WinRAR:** Одна з найпопулярніших програм, що підтримує створення та розпакування RAR та ZIP архівів, а також розпакування CAB, ARJ, LZH, TAR, GZ, ACE, UUE, BZ2, JAR, ISO, 7Z, Z архівів. Відома своїм високим ступенем стиснення для RAR-формату та функціями відновлення пошкоджених архівів.
    *   **7-Zip:** Безкоштовний архіватор з відкритим вихідним кодом. Підтримує власний формат 7z з дуже високим ступенем стиснення, а також ZIP, GZIP, BZIP2, TAR та інші формати. Є дуже ефективним і часто перевершує інші архіватори за ступенем стиснення.
    *   **WinZip:** Комерційний архіватор, який був одним з перших і найвідоміших для Windows. Підтримує ZIP, ZIPX, RAR, 7z, TAR, GZ та інші формати. Пропонує розширені функції, такі як шифрування, інтеграція з хмарними сервісами, конвертація PDF.
    *   **PeaZip:** Безкоштовний архіватор з відкритим вихідним кодом, що підтримує понад 150 форматів архівів, включаючи 7Z, ARC, BZ2, GZ, PAQ, PEA, RAR, TAR, WIM, XZ, ZIP, ZIPX. Має інтуїтивно зрозумілий інтерфейс та розширені функції безпеки.

### 4.6. Використання стиснення та архівування для резервування даних та в системному адмініструванні

**Резервування даних:** Стиснення та архівування є фундаментальними компонентами стратегій резервного копіювання даних:

*   **Економія місця:** Стиснення значно зменшує обсяг даних, які потрібно зберігати, що дозволяє зберігати більше резервних копій на тому ж носії або зменшити витрати на хмарне сховище.
*   **Зменшення часу передачі:** Менші файли швидше передаються по мережі, що прискорює процес створення резервних копій на віддалені сервери або хмарні сховища.
*   **Цілісність даних:** Архівування об'єднує багато файлів в один, що спрощує управління та перевірку цілісності резервної копії. Деякі формати архівів (наприклад, RAR, 7z) мають вбудовані механізми відновлення даних.
*   **Збереження метаданих:** Такі інструменти, як `tar`, зберігають права доступу, власників, часові мітки та структуру каталогів, що є критично важливим для точного відновлення системи.

**Інші задачі системного адміністрування:**

*   **Розповсюдження програмного забезпечення:** Програмне забезпечення та оновлення часто пакуються у стиснені архіви (наприклад, `.tar.gz`, `.zip`, `.deb`, `.rpm`) для ефективної дистрибуції та встановлення.
*   **Управління лог-файлами:** Системні лог-файли можуть швидко зростати. Адміністратори часто використовують стиснення (наприклад, `logrotate` з `gzip`) для архівування старих логів, щоб заощадити місце, зберігаючи при цьому можливість їх аналізу.
*   **Передача файлів:** При передачі великої кількості файлів або великих файлів між серверами або користувачами, їх стиснення та архівування значно прискорює процес та зменшує навантаження на мережу.
*   **Управління конфігураціями:** Збереження конфігураційних файлів у стиснених архівах дозволяє легко відстежувати зміни, створювати резервні копії та відновлювати попередні стани.
*   **Звільнення дискового простору:** Адміністратори регулярно використовують стиснення для звільнення місця на переповнених дисках, особливо для рідко використовуваних даних.
*   **Аудит та аналіз:** Стиснені архіви можуть бути використані для зберігання даних аудиту або знімків системи для подальшого аналізу без використання великого обсягу пам'яті.

### 4.7. Призначення директорії файлу `/dev/null`

`/dev/null` - це спеціальний файл у Unix-подібних операційних системах (включаючи Linux), який називають "нульовим пристроєм" або "чорною дірою".

**Призначення:**
*   **Відкидання виводу:** Будь-які дані, записані в `/dev/null`, відкидаються і не зберігаються. Це корисно, коли потрібно виконати команду, але її вивід (стандартний вивід або помилки) не потрібен і його потрібно ігнорувати, щоб він не засмічував термінал або не записувався у файл.
    *   **Приклад:** `command > /dev/null` (відкидає стандартний вивід)
    *   **Приклад:** `command 2> /dev/null` (відкидає стандартний вивід помилок)
    *   **Приклад:** `command &> /dev/null` (відкидає обидва потоки)

*   **Надання порожнього вводу:** Читання з `/dev/null` негайно повертає кінець файлу (EOF), що еквівалентно порожньому вводу. Це може бути використано для команд, які очікують ввід, але в даному випадку він не потрібен.
    *   **Приклад:** `tar -xvf archive.tar < /dev/null` (якщо `tar` очікує ввід, але його немає)

По суті, `/dev/null` діє як віртуальний пристрій, який приймає будь-які дані, але нічого не зберігає, і повертає порожній результат при читанні. Це потужний інструмент для контролю потоків вводу/виводу в сценаріях Bash та командному рядку.
