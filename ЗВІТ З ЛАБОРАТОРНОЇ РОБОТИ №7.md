## 2. ЗАВДАННЯ ПОПЕРЕДНЬОЇ ПІДГОТОВКИ

### 2.1 Словник базових термінів (English)
| Term | Definition |
| :--- | :--- |
| **Icon** | A visual symbol representing an action, object, or concept in a user interface to improve navigation and reduce cognitive load. |
| **Outline Style** | Icon style using strokes without fill, creating a lightweight, minimal appearance suitable for modern UI. |
| **Filled Style** | Icon style with solid fill, providing stronger visual weight and better visibility for important actions. |
| **Duotone** | Icon style using two colors or tones to add depth, dimension, and visual interest to interface elements. |
| **Bounding Box** | An invisible square frame (e.g., 24×24 px) that contains an icon, ensuring consistent sizing and alignment across a set. |
| **Optical Alignment** | Visual adjustment of elements to appear balanced to the human eye, rather than relying solely on mathematical centering. |
| **Safe Zone** | The inner area within a bounding box where the actual icon graphic resides, typically with 2px padding for visual consistency. |

### 2.2 Відповіді на теоретичні питання
1. **Які стилі іконок існують та де вони використовуються?**
   - **Outline:** Лінійні, без заповнення. Використовуються в сучасних мінімалістичних інтерфейсах та веб-додатках.
   - **Filled:** Суцільне заповнення. Використовуються для активних станів та важливих дій.
   - **Duotone:** Два кольори/відтінки. Використовуються для акцентних елементів та брендованих інтерфейсів.
   - **Glyph:** Компактні символи. Використовуються в системних іконках та мобільних додатках.

2. **Що таке оптичне вирівнювання і чому воно важливе?**
   - Це процес візуального налаштування позиції елементів для досягнення балансу людським оком. Це важливо, бо різні геометричні форми сприймаються по-різному навіть при однакових математичних розмірах.

3. **Популярні відкриті бібліотеки іконок:**
   - **Lucide Icons:** https://lucide.dev/ (1000+ векторних SVG-іконок).
   - **Heroicons:** https://heroicons.com/ (створені командою Tailwind CSS).
   - **Feather Icons:** https://feathericons.com/ (класична мінімалістична бібліотека).

## 3. ПРАКТИЧНЕ ЗАВДАННЯ №1: АНАЛІЗ І ПІДБІР ІКОНОК

### 3.1 Принципи підбору іконок для Deadline Tracker:
- **Єдність стилю:** Усі іконки мають виглядати як єдиний набір.
- **Узгоджена товщина ліній:** Вибрано 1.5px stroke.
- **Інтуїтивність:** Користувач має розуміти дію без тексту.
- **Bounding Box:** Однакові квадратні фрейми (24×24 px).

### 3.2 Обґрунтування вибору бібліотеки:
Для проєкту **Deadline Tracker** з шрифтом **Sans-serif (Grotesque)** обрано бібліотеку **Lucide Icons** у стилі **Outline**.
- Лінійні іконки створюють відчуття легкості.
- Чудово поєднуються з гротесками.
- Мають сучасний діловий вигляд.

## 5. ВІДПОВІДІ НА КОНТРОЛЬНІ ЗАПИТАННЯ

1. **Чому важливо використовувати SVG-формат для іконок?**
   - **Масштабованість:** Чіткість на будь-якому розмірі.
   - **Малий розмір:** Швидке завантаження.
   - **Кастомізація:** Зміна кольору та розміру через код.
   - **SEO:** Текстовий вміст індексується.

2. **У яких випадках іконки повинні супроводжуватися текстом?**
   - При використанні неоднозначних символів.
   - У головній навігації для покращення UX.
   - Для забезпечення доступності (скрінрідери).

3. **Що таке "Safe zone" всередині контейнера іконки?**
   - Це внутрішня область bounding box (зазвичай 20x20 всередині 24x24), де розташовується графіка, залишаючи відступи для візуального простору.

4. **Чим відрізняється іконка в стилі "Glyph" від іконки "Outline"?**
   - **Glyph:** Суцільне заповнення, більша візуальна вага.
   - **Outline:** Тільки контур, легший та сучасніший вигляд.
