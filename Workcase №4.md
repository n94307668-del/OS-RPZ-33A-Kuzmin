## 1. Основні поняття

### 1.1 Визначення поняття «пакет»
**Пакет** — це архівний файл, що містить усі необхідні компоненти для встановлення програмного забезпечення або бібліотеки. У системах Linux пакет зазвичай включає:
- Бінарні виконувані файли;
- Конфігураційні файли;
- Інформацію про залежності (інші пакети, необхідні для роботи);
- Документацію;
- Метадані (версія, автор, опис).

> Пакети використовують специфічні формати залежно від дистрибутива. Найпоширенішими є **.deb** (Debian/Ubuntu) та **.rpm** (Red Hat/Fedora/SUSE).

### 1.2 Визначення поняття «репозиторій»
**Репозиторій (сховище)** — це централізоване місце зберігання, де розміщуються та підтримуються пакети програмного забезпечення. Репозиторії дозволяють менеджеру пакетів автоматично шукати, завантажувати та встановлювати програми.

| Тип репозиторію | Опис |
| :--- | :--- |
| **Офіційні** | Підтримуються розробниками дистрибутива, гарантують стабільність. |
| **Сторонні (PPA)** | Підтримуються розробниками ПЗ або спільнотою для отримання нових версій. |
| **Локальні** | Зберігаються безпосередньо на комп'ютері користувача. |

### 1.3 Огляд існуючих менеджерів пакетів у Linux
Різні дистрибутиви використовують власні інструменти для керування програмним забезпеченням.

| Менеджер пакетів | Дистрибутиви | Основні можливості |
| :--- | :--- | :--- |
| **APT** | Debian, Ubuntu, Mint | Вирішення залежностей, автоматичні оновлення, величезна база пакетів. |
| **DNF / YUM** | Fedora, RHEL, CentOS | Покращене вирішення залежностей, історія транзакцій, підтримка плагінів. |
| **Pacman** | Arch Linux, Manjaro | Простий синтаксис, підтримка AUR, модель оновлення Rolling Release. |
| **Zypper** | openSUSE | Швидкість, надійність, відмінна робота з залежностями. |
| **Snap / Flatpak** | Універсальні | Контейнеризовані додатки, ізоляція (sandbox), працюють у будь-якому дистрибутиві. |

## 2. Менеджер пакетів APT

Мій дистрибутив (Ubuntu) використовує менеджер пакетів **APT (Advanced Package Tool)**. Це потужний інструмент, який автоматизує процес встановлення, налаштування та видалення програм.
![alt text](image-10.png)
### 2.1 Основні команди для роботи з APT

| Дія | Команда |
| :--- | :--- |
| **Пошук пакетів** | `apt search <назва>` |
| **Оновлення списків пакетів** | `sudo apt update` |
| **Встановлення програми** | `sudo apt install <назва>` |
| **Видалення програми** | `sudo apt remove <назва>` |
| **Повне видалення (з конфігами)** | `sudo apt purge <назва>` |
| **Оновлення встановлених програм** | `sudo apt upgrade` |
| **Перегляд інформації про пакет** | `apt show <назва>` |
| **Очищення кешу** | `sudo apt clean` |
| **Видалення непотрібних залежностей** | `sudo apt autoremove` |

### 2.2 Приклади встановлення пакетів

#### Встановлення медіаплеєра (VLC)
VLC — це популярний програвач з відкритим кодом, що підтримує майже всі формати відео та аудіо.
```bash
sudo apt update
sudo apt install vlc
```

#### Встановлення середовища програмування (Python)
Для вивчення мови Python необхідно встановити інтерпретатор та інструменти розробки:
```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv build-essential
```
Це встановить Python 3, менеджер пакетів pip та інструменти для створення віртуальних середовищ.

## 3. Встановлення програм у графічному середовищі

1.  **Ubuntu Software (Центр програм):** Аналог магазинів додатків (як App Store чи Google Play). Дозволяє шукати програми за категоріями, читати відгуки та встановлювати їх одним натисканням кнопки.
2.  **Synaptic:** Більш просунутий графічний менеджер, який дозволяє детально керувати кожним пакетом, вибирати версії та керувати репозиторіями.
3.  **GDEBI:** Утиліта для встановлення окремих `.deb` файлів, завантажених з інтернету, яка автоматично підтягує необхідні залежності.
