### 2. Відповіді на контрольні запитання

#### 4.1. Розкрийте поняття "псевдо файлової системи", для чого воно потрібне системі?

**Псевдо файлова система (pseudo filesystem)** - це віртуальна файлова система, яка існує тільки в оперативній пам'яті комп'ютера і не зберігається на жорсткому диску. Вона виглядає як звичайна ієрархія файлів та директорій, але насправді є інтерфейсом для доступу до системної інформації ядра.

**Призначення псевдо файлових систем:**
- Надають доступ до інформації про процеси, пам'ять, пристрої та конфігурацію ядра
- Дозволяють переглядати та іноді змінювати параметри роботи системи в реальному часі
- Стандартизований інтерфейс взаємодії з ядром через файлову систему
- Приклади: `/proc` (інформація про процеси), `/sys` (інформація про пристрої), `/dev` (пристрої)

#### 4.2. Чому користувачі не так часто звертаються на пряму до каталогу /proc, яким чином з нього можна отримати інформацію?

Користувачі рідко звертаються безпосередньо до `/proc` з кількох причин:
- Інформація в `/proc` представлена у вигляді числових файлів та директорій, що важко читати
- Структура `/proc` складна і вимагає глибоких знань про ядро
- Існують спеціальні команди, які зчитують та форматують дані з `/proc` у зручному вигляді

**Команди для отримання інформації з `/proc`:** `top`, `free`, `mount`, `umount`, `ps`, `cat /proc/[файл]`

#### 4.3. Яке призначення файлів /proc/cmdline, /proc/meminfo та /proc/modules?

- **`/proc/cmdline`** - містить інформацію, яка була передана ядру при його запуску (параметри командного рядка, спеціальні інструкції)
- **`/proc/meminfo`** - містить детальну інформацію про використання оперативної пам'яті ядром (загальна пам'ять, вільна, буфери, кеш тощо)
- **`/proc/modules`** - містить список модулів ядра, які зараз завантажені в систему для додавання додаткової функціональності

#### 4.4. Яке призначення команди free?

Команда `free` використовується для відображення інформації про використання оперативної пам'яті в системі. Вона показує:
- Загальний обсяг оперативної пам'яті
- Використану пам'ять
- Вільну пам'ять
- Пам'ять у буферах та кеші
- Інформацію про swap-пам'ять

З параметром `-s` можна оновлювати виведення через задану кількість секунд (наприклад, `free -s 10`).

#### 4.5. Для чого потрібні лог-файли, наведіть приклади їх застосування?

**Лог-файли (файли журналів) потрібні для:**
- Діагностики проблем та усунення несправностей
- Відстеження спроб несанкціонованого доступу
- Моніторингу роботи системи та служб
- Аудиту дій користувачів
- Аналізу помилок та їх причин

**Основні лог-файли в `/var/log`:**
- `boot.log` - повідомлення під час завантаження системи
- `messages/syslog` - повідомлення ядра та системних процесів
- `auth.log/secure` - повідомлення про автентифікацію та авторизацію
- `dmesg` - повідомлення ядра під час завантаження
- `cron` - повідомлення планувальника завдань

#### 4.6. Яке призначення файлу /var/log/dmesg?

Файл `/var/log/dmesg` містить повідомлення ядра (kernel messages), які були згенеровані під час завантаження системи. Ці повідомлення включають інформацію про:
- Виявлене обладнання та драйвери
- Ініціалізацію пристроїв
- Помилки та попередження під час завантаження
- Параметри ядра

Для перегляду використовується команда `dmesg`, яка показує вміст кільцевого буфера ядра.

#### 4.7. Для чого розроблено FHS?

**FHS (Filesystem Hierarchy Standard)** - це стандарт ієрархії файлової системи, розроблений для:
- Уніфікації структури директорій у різних дистрибутивах Linux
- Визначення призначення кожної системної директорії
- Забезпечення сумісності між різними системами
- Полегшення адміністрування та розробки ПЗ

FHS класифікує директорії за ознаками: shareable/unshareable (можна/не можна розшарювати по мережі) та static/variable (статичні/змінні файли).

#### 4.8. Які основні команди є у Linux для перегляду та конфігурації мережі?

**Основні команди для роботи з мережею:**
- `ifconfig` - перегляд та налаштування мережевих інтерфейсів (застаріла)
- `ip addr show` / `ip link` - сучасна альтернатива ifconfig
- `route` - перегляд та налаштування таблиці маршрутизації
- `ping` - перевірка доступності хоста
- `netstat` - інформація про мережеві з'єднання
- `ss` - сучасна альтернатива netstat для перегляду сокетів
- `dig` / `host` - робота з DNS (перевірка записів)
- `ssh` - віддалене підключення до інших систем
- `traceroute` - відстеження маршруту пакетів

## Завдання 3: Практичні завдання

### 1. Команда cat

Команда `cat` (concatenate) призначена для виведення вмісту файлів, їх об'єднання та створення нових файлів.

**Основні можливості:**
- Перегляд вмісту файлу: `cat filename.txt`
- Створення файлу: `cat > newfile.txt` (потім ввести текст, Ctrl+D для завершення)
- Об'єднання файлів: `cat file1.txt file2.txt > combined.txt`
- Додавання вмісту до файлу: `cat >> existing.txt`
- Перенаправлення: `cat file.txt | grep pattern`

**Корисні параметри:**
- `-n` - нумерація рядків
- `-A` - відображення недрукованих символів
- `-s` - видалення порожніх рядків
- `-b` - нумерація непорожніх рядків
- `-E` - відображення символу $ в кінці рядка

### 2. Команда dig

Команда `dig` (Domain Information Groper) використовується для запитів до DNS-серверів та діагностики DNS.

**Основні можливості:**
- Отримання IP-адреси за доменним іменем: `dig example.com`
- Зворотний DNS-запит: `dig -x 192.168.1.1`
- Запит певного типу запису: `dig example.com MX` (поштові записи)
- Використання конкретного DNS-сервера: `dig @8.8.8.8 example.com`
- Короткий вивід: `dig +short example.com`

**Приклади використання:**
- `dig localhost.localdomain` - запит інформації про локальний домен
- `dig cserver.example.com` - запит інформації про сервер
- `dig -x 192.168.1.2` - зворотний запит для IP-адреси

### 3. Команда netstat

Команда `netstat` (network statistics) використовується для відображення мережевих з'єднань, таблиць маршрутизації, статистики інтерфейсів тощо.

**Основні можливості:**
- Перегляд всіх з'єднань: `netstat -a`
- Перегляд TCP-з'єднань: `netstat -t`
- Перегляд listening портів: `netstat -l`
- Перегляд з числовими адресами: `netstat -n`
- Перегляд статистики: `netstat -s`

**Приклади використання:**
- `netstat -tl` - показує TCP listening сокети
- `netstat -tln` - показує TCP listening сокети з числовими адресами
- `netstat -an | grep :80` - пошук з'єднань на порту 80

## Відповіді на контрольні запитання

### 1. Як пов'язані між собою команди cat та tac?

Команди `cat` та `tac` є протилежними за функціональністю:
- `cat` (concatenate) - виводить вміст файлу в прямому порядку (з початку до кінця)
- `tac` (cat навпаки) - виводить вміст файлу у зворотному порядку (з кінця до початку)
- `tac` корисна для перегляду лог-файлів, де найновіші записи потрібні першими

### 2. Що робить команда ss?

Команда `ss` (socket statistics) використовується для відображення статистики сокетів. Це сучасна альтернатива netstat, яка:
- Показує інформацію про мережеві з'єднання
- Підтримує всі основні типи сокетів (TCP, UDP, UNIX тощо)
- Працює швидше за netstat
- Надає більше інформації про з'єднання
- Дозволяє фільтрувати результати за різними критеріями

### 3. В чому відмінність між командами ps --forest та pstree?

Обидві команди показують ієрархію процесів, але мають відмінності:
- `ps --forest` - показує процеси у вигляді дерева з використанням ASCII-графіки, включаючи детальну інформацію про кожен процес (PID, CPU, MEM тощо)
- `pstree` - спеціалізована команда для відображення тільки дерева процесів у більш компактному та читабельному вигляді
- `pstree` за замовчуванням не показує PID, але можна додати `-p` для їх відображення
- `pstree` краще підходить для швидкого огляду структури процесів

### 4. У яких каталогах зберігаються налаштування системи?

Налаштування системи зберігаються в каталозі `/etc` (editable text configuration). Цей каталог містить:
- Конфігураційні файли системи
- Налаштування системних служб
- Файли автентифікації (passwd, shadow, group)
- Мережеві налаштування (network/, resolv.conf)
- Налаштування планувальника завдань (cron.*)

### 5. У яких каталогах можна знайти встановлені в системі програми, доступні для користувача?

Програми, доступні для користувача, знаходяться в:
- `/bin` - базові команди (cat, ls, cp, mv тощо)
- `/usr/bin` - основні програми для користувачів
- `/usr/local/bin` - локально встановлені програми
- `/opt` - додаткові програми (часто комерційні)
- `/home/user/bin` - програми конкретного користувача

### 6. У яких каталогах можна знайти встановлені системні програми і програми призначені для виконання суперкористувачем?

Системні програми та програми для root знаходяться в:
- `/sbin` - системні програми (fdisk, fsck, reboot, shutdown тощо)
- `/usr/sbin` - додаткові системні програми
- `/usr/local/sbin` - локально встановлені системні програми
- Ці програми потребують прав root для виконання

### 7. Поясніть призначення команд ping, ifconfig, traceroute.

**ping** - перевіряє доступність мережевого хоста шляхом відправки ICMP-запитів та очікування відповідей. Використовується для діагностики мережевого з'єднання та вимірювання затримки.

**ifconfig** (interface configuration) - відображає та налаштовує мережеві інтерфейси. Показує IP-адресу, маску підмережі, MAC-адресу, статус інтерфейсу. Застаріла команда, замінена на `ip addr`.

**traceroute** - відстежує маршрут пакетів від джерела до призначення, показуючи всі проміжні маршрутизатори. Корисна для діагностики проблем на шляху до хоста.

### 8. Як називаються мережеві інтерфейси в Linux?

Традиційні назви мережевих інтерфейсів в Linux:
- `eth0`, `eth1`, ... - Ethernet інтерфейси (провідні)
- `wlan0`, `wlan1`, ... - бездротові інтерфейси Wi-Fi
- `lo` - loopback інтерфейс (локальна петля, 127.0.0.1)

В сучасних дистрибутивах використовуються передбачувані імена (Predictable Network Interface Names):
- `enp0s3` - Ethernet PCI Express
- `ens33` - Ethernet на віртуальній машині VMware
- `wlp2s0` - бездротовий інтерфейс PCI

### 9. Як за допомогою команди ifconfig вивести параметри тільки одного мережевого інтерфейсу (наприклад, eth1), а не всіх?

Для виведення параметрів конкретного інтерфейсу потрібно вказати його ім'я як аргумент команди:
```bash
ifconfig eth1
```

Це покаже інформацію тільки для інтерфейсу eth1, включаючи:
- IP-адресу та маску підмережі
- MAC-адресу
- Статус інтерфейсу (UP/DOWN)
- Статистику пакетів (отримано/передано)
